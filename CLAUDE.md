# Live-Data — Upstox WebSocket V3 Market Feed + F&O Analysis

Streams/fetches live Indian market data from Upstox and analyses it (technicals +
option chain + latest news + historical precedent) to surface risk-managed F&O trade
ideas. Risk-first decision-support — NOT guaranteed profit.

## Project structure
```
Live-Data/
├── README.md            # human daily-use guide + full strategy
├── CLAUDE.md            # this file (auto-loaded each session)
├── requirements.txt
├── .env                 # UPSTOX_API_KEY / _API_SECRET / _REDIRECT_URI (gitignored)
├── .access_token        # generated daily by auth/get_token (gitignored)
├── trade_journal.csv    # generated by journal (gitignored)
├── src/                 # all PERMANENT Python scripts (run from project root)
├── docs/                # TRADING_RULES.md, STRATEGIES.md, PIPELINE.md, WORKFLOW.md
├── config/              # positions.example.json (copy -> positions.json)
├── data/                # market data output (gitignored)
├── scratch/             # throwaway one-off analysis scripts (gitignored)
└── .claude/agents/      # the agents
```
Run scripts from the project root with the venv Python, e.g. `.\.venv\Scripts\python.exe src/auth.py`. Python 3.14. All scripts anchor their file paths to the project root, so CWD doesn't matter.

**Throwaway scripts:** if an agent or analysis step needs a one-off/ad-hoc Python script (not a permanent tool), write it to `scratch/` — NEVER to the project root or `src/`. `scratch/` is gitignored and safe to empty anytime; `src/` is reserved for the permanent, documented scripts listed below.

## Scripts (`src/`)
- `src/run_pipeline.py ["<redirect-url>"]` — one-command DATA stage: token (optional) → `market_snapshot` → readiness report. Entry point for the orchestrated pipeline (see `docs/PIPELINE.md`).
- `src/auth.py` — interactive daily Upstox OAuth2 login; saves `.access_token`.
- `src/get_token.py "<code-or-redirect-url>"` — non-interactive token exchange (paste-the-URL flow).
- `src/market_snapshot.py` — fresh REST snapshot → `data/snapshot_*.json`: market status, intraday (5m) + daily candles, nearest-expiry option chain (OI/IV/greeks), news.
- `src/streamer.py` — WebSocket V3 tick stream → `data/market_data_*.json` every 5s.
- `src/backtest.py` — validate a directional setup on historical candles (win-rate/expectancy/PF/drawdown). Underlying-signal only, not option P&L.
- `src/monitor_positions.py` — watch open trades in `config/positions.json` vs stop/target/time-stop via live LTP (alert-only).
- `src/journal.py add|stats` — log trades → `trade_journal.csv`; compute real win-rate/expectancy/cost-drag.
- `src/cleanup_data.py` — safely delete generated data in `data/` (+ optional `__pycache__`). Flags: `--keep-latest N`, `--older-than N`, `--pycache`, `--dry-run`. Path-guarded to `data/` only.

## Docs (`docs/`)
- `docs/TRADING_RULES.md` — master rulebook: hard risk rules, 9-point pre-trade checklist, regime/IV playbook, strategy matrix, "do not trade" conditions. Trade agents must obey it.
- `docs/STRATEGIES.md` — options strategy playbook (long/debit/credit spreads, straddle/strangle, iron condor/fly, calendar, expiry/0DTE).
- `docs/PIPELINE.md` — orchestrated pipeline stage graph + rules. `docs/WORKFLOW.md` — full system diagram (Mermaid + ASCII).

## Agents (`.claude/agents/`)
- `news-scanner` — latest same-day news/cues → directional bias + key levels.
- `fno-strategist` — builds the F&O trade card per `docs/TRADING_RULES.md` + `docs/STRATEGIES.md`; scores the 9-point checklist.
- `risk-manager` — independent gatekeeper; re-checks a proposed trade vs the rulebook → APPROVE / APPROVE-WITH-CHANGES / VETO.
- `backtester` — validates a setup's historical edge via `src/backtest.py` before it's trusted.
- `position-monitor` — watches open trades vs stop/target/time-stop (uses `src/monitor_positions.py`); alert-only.
- `trade-journal` — logs trades and reports real win-rate/expectancy/cost-drag (uses `src/journal.py`).
- `market-analyst` — general technical analysis engine for any instrument/category.
- `data-janitor` — safely cleans up generated data on request (uses `src/cleanup_data.py`); only `data/` + `__pycache__`, never source/creds/token.

## Daily pipeline (ONE-SHOT, AUTONOMOUS — suggestion-only; freshness enforced)
Full stage graph + rules: `docs/PIPELINE.md`. **SUGGESTION-ONLY: never place/modify/cancel orders** (TRADING_RULES §0) — output is always a suggestion the user executes manually.

**Flow:** user pastes the login URL → **ask intent ONCE** (category e.g. F&O intraday / positional, underlying, capital, any bullish/bearish lean) → then run the ENTIRE pipeline autonomously and silently, with **NO intermediate approvals or stage-by-stage check-ins**:
  `run_pipeline.py` (token → snapshot) → `news-scanner` ∥ `backtester` → `fno-strategist` → `risk-manager` (**INTERNAL** check) → present **ONLY the final suggestion** (a trade card, or "stay flat").
- Fold the risk-manager verdict into the single final card; do NOT surface it as a separate APPROVE/VETO step.
- **Interrupt the user ONLY if something errors/breaks** (token/data/API failure) or intent is genuinely ambiguous.
- After the user says they entered: `position-monitor` + `trade-journal` (also suggestion/alert-only).
The main session orchestrates (sub-agents can't spawn sub-agents); run independent stages concurrently.

## Important constraints (do not violate)
- **Suggestion-only — never places/modifies/cancels orders.** No execution code path; the user places every order manually. One-shot flow: ask intent once, then deliver only the final suggestion (no mid-pipeline approvals); interrupt only on error.
- **No guaranteed win rates or ROI.** ~9/10 retail F&O traders lose (SEBI). Risk ≤2% of capital/trade; size from the stop-loss; always include SL + sizing + max loss + own-risk reminder. "No clean setup → stay flat" is valid.
- Requires fresh live data + latest news + historical context; flag low confidence if any are stale/missing.
- Outside market hours: only last-close/snapshot data → lower-confidence analysis.
- Claude can't do the browser login (OAuth + 2FA) — the user does that one step and pastes the redirect URL.
